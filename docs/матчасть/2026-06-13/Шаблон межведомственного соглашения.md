# Шаблон межведомственного соглашения об обмене данными «Цифрового двойника России»

> **Дата:** 2026-06-09
> **Документ-ответ на:** [issue #40](https://github.com/PavelChurkin/resource-based-economy-Article/issues/40)
> **Связанный PR:** [#41](https://github.com/PavelChurkin/resource-based-economy-Article/pull/41)
> **Назначение:** типовая форма межведомственного соглашения об информационном взаимодействии для ведомственного контура двойника

Документ отвечает на требование [issue #40](https://github.com/PavelChurkin/resource-based-economy-Article/issues/40): *«…составь Шаблон межведомственного соглашения»*. Он оформляет в виде готового к адаптации шаблона организационную схему, описанную в [FAQ](Чаво.md), вопрос 5, и развивает её до уровня **рабочего документа** с приложениями.

> ⚠️ **Юридическая оговорка.** Это методический шаблон, а не подписанный нормативный акт. Соглашение заключается **государственным оператором** (федеральный или региональный орган), а не инициатором проекта. До подписания обязательна правовая экспертиза (ФЗ-149, ФЗ-152, ФЗ-187, отраслевые законы, постановления Правительства РФ). Инициатор без полномочий **не запрашивает данные самостоятельно** — он предлагает «создать рамку и оператора» ([FAQ](Чаво.md), §5.6).

## 1. Когда применяется этот шаблон

| Контур | Применимость шаблона |
|--------|----------------------|
| Открытый | Не нужен — данные открыты, соглашение не требуется |
| **Ведомственный** | **Основная область применения этого шаблона** |
| Инфраструктурный | Используется [Шаблон соглашения с оператором инфраструктуры](Шаблон%20соглашения%20с%20оператором%20инфраструктуры.md) (обмен ограничениями) |
| Закрытый | Только в режиме уполномоченного органа, отдельным регламентом |

Шаблон вступает в работу после появления **государственного оператора пилота** — ключевой блокирующей зависимости проекта ([отчёт №2](Отчёт_анализа_2026-06-01_23-57-56_UTC.md), §2, риск №1).

#### Статья 2. Термины и сокращения

2.1. **Контур** — уровень чувствительности данных и режим доступа (открытый, ведомственный, инфраструктурный, закрытый, исследовательский).

3.1. Перечень наборов данных приведён в Приложении А «Паспорт набора данных» по каждому набору.
3.2. Изменение перечня оформляется дополнительным соглашением.
3.3. Персональные данные передаются **только** в обезличенном/агрегированном виде, если иное прямо не предусмотрено законом и отдельным регламентом.

## 3. Приложения к соглашению

### Приложение А. Паспорт набора данных

Воспроизводит структуру паспорта качества ([Функционал SaaS](Функционал%20SaaS.md), §5.2; [отчёт №1](Отчёт_анализа_2026-05-21_17-52-10_UTC.md), §6.1):

```yaml
dataset_passport:
  id: "<идентификатор набора>"
  owner: "<владелец данных>"
  source: "<источник>"
  license: "<условия использования>"
  update_frequency: "<сутки|неделя|месяц|квартал|год>"
  classifier_alignment: "<ОКТМО/ОКВЭД2/ОКЗ/...>"
  geometry_crs: "<EPSG:4326 / ГСК-2011 / нет>"
  pii_status: "<none|aggregated|anonymized|pseudonymized>"
  k_anonymity: "<значение k или н/п>"
  known_limitations: "<ограничения и оговорки о неполноте>"
  contour: "departmental"
  validators: "<кто проверяет>"
  signed_by: "<ответственное лицо>"
  certificate_version: "<версия>"
  signed_at: "<дата>"
```

## 4. Чек-лист готовности к подписанию

- [ ] Определён государственный оператор (без него соглашение не заключается);
- [ ] Выбран правовой режим (ФЗ-149/152/187 + отраслевые акты);
- [ ] Заполнены паспорта по каждому набору (Приложение А, 100%);
- [ ] Зафиксирован уровень обезличивания и k-анонимность для чувствительных наборов;
- [ ] Определены каналы, форматы и периодичность обмена;
- [ ] Описан регламент доступа и журналирования;
- [ ] Описан порядок реагирования на инциденты;
- [ ] Распределены финансовые обязательства;
- [ ] Заданы срок, пересмотр, расторжение и судьба данных при выходе;
- [ ] Пройдена правовая экспертиза.
