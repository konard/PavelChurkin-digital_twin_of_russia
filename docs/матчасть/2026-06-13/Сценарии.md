# Сценарии «Цифрового двойника России»

## Назначение документа

Документ собирает максимально полный перечень сценариев применения «Цифрового двойника России». Перечень составлен для использования в бизнес-плане и в проектной документации: каждый сценарий формулируется через пользователя, задачу, входные слои данных, ожидаемый результат и контур доступа.

Документ дополняет [Описание](Описание.md), [MVP](MVP.md), [Тезисы](Тезисы.md) и [Чаво](Чаво.md). В отличие от них, здесь не описывается архитектура или правовая рамка, а перечисляются конкретные сценарии — то, ради чего система разворачивается.

Сценарии сгруппированы по тематическим доменам. Для каждого домена указан принцип отбора, чтобы перечень можно было расширять при поступлении новых задач от ведомств, регионов и отраслей.

---

## Условные обозначения

Для каждого сценария указываются:

| Поле | Содержание |
|------|------------|
| Пользователь | Кто инициирует сценарий и принимает решение по результату |
| Задача | Управленческий или операционный вопрос, на который отвечает сценарий |
| Слои данных | Минимальный набор слоёв и реестров, необходимых для расчёта |
| Результат | Что получает пользователь по итогу: карта, отчёт, сравнение, прогноз |
| Контур | Открытый, ведомственный, инфраструктурный, закрытый или исследовательский |
| Сложность | Низкая, средняя или высокая — оценка трудоёмкости реализации в MVP |

Описание контуров доступа приведено в разделе «Тезис 3» документа [Тезисы](Тезисы.md) и в [Примерах обработки информации для контуров](Примеры%20обработки%20информации%20для%20контуров.md).

## Сводка по сложности

Для использования в бизнес-плане сценарии распределены по сложности реализации в MVP.

| Сложность | Признаки | Примеры |
|-----------|----------|---------|
| Низкая | 1–2 источника данных, существующие методики, открытый контур | Карта культурной инфраструктуры, общественные пространства, поиск образовательных программ, карта услуг |
| Средняя | 3–6 источников данных, простые модели, открытый и ведомственный контур | Региональный паспорт, подбор площадки, карта кадровых дефицитов, мониторинг качества воздуха |
| Высокая | 7+ источников данных, сценарное моделирование, инфраструктурный и закрытый контур | Граф зависимостей инфраструктуры, мобилизационное планирование, климатические сценарии, межотраслевая кооперация |

В MVP целесообразно реализовать пакет из 6–10 сценариев низкой и средней сложности по 2–3 пилотным регионам. Сценарии высокой сложности — после расширения инфраструктурного и закрытого контуров.

## Сводка по контурам доступа

Перечень сценариев распределён по контурам доступа. Это помогает планировать поэтапное наполнение системы и согласование с уполномоченными органами.

| Контур | Примерное число сценариев | Особенности |
|--------|----------------------------|-------------|
| Открытый | 30+ | Готовы к запуску в первой очереди MVP |
| Ведомственный | 50+ | Требуют соглашений с владельцами данных |
| Инфраструктурный | 10+ | Требуют участия операторов критической инфраструктуры |
| Закрытый | 10+ | Требуют согласования с уполномоченными органами |
| Исследовательский | 10+ | Требуют обезличенных или синтетических наборов |

Один сценарий может работать в нескольких контурах одновременно с разной глубиной детализации.

## Использование документа в бизнес-плане

Документ предназначен для следующих задач:

1. **Сборка дорожной карты MVP.** Из списка выбираются 6–10 сценариев низкой и средней сложности, которые формируют пакет первой очереди.
2. **Оценка трудозатрат.** По каждому сценарию указаны слои данных и сложность — на их основе формируются оценки команды и сроков.
3. **Формирование пакетов услуг.** Сценарии группируются в коммерческие или государственные пакеты услуг (см. список услуг в [Описании](Описание.md)).
4. **Подготовка обращений в ведомства.** К каждому сценарию подбираются ведомства-источники данных и потенциальные ведомства-заказчики.
5. **Подготовка пилотов в регионах.** В регионе выбираются сценарии, которые соответствуют его профилю (промышленный, аграрный, транспортный, городской).
6. **Подготовка предложений по правовому режиму.** Сценарии в закрытых контурах формируют требования к регламентам и доступу.
7. **Подготовка предложений по международной интеграции.** Сценарии открытого международного слоя формируют рамку взаимодействия с дружественными странами.

## Принципы расширения перечня

Перечень не является закрытым. Принципы добавления новых сценариев:

- сценарий формулируется через пользователя, задачу, слои данных, результат и контур;
- сценарий должен закрывать конкретную потребность государства, региона, отрасли, бизнеса, науки или гражданина;
- сценарий не должен раскрывать чувствительные данные в открытом контуре;
- сценарий должен опираться на уже доступные или потенциально доступные источники данных;
- сценарий должен иметь измеримый результат (карта, отчёт, прогноз, подбор, сравнение);
- сценарий должен быть согласован с принципами безопасности и контурами доступа;
- сценарий должен быть проверяемым на пилоте — без проверки сценарий считается гипотезой, а не подтверждённым кейсом.

При выявлении новых сценариев документ обновляется и сохраняет историю изменений в системе контроля версий.
