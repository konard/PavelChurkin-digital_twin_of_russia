# Дорожная карта по версиям «Цифрового двойника России»

> **Назначение.** Документ определяет дорожную карту по версиям продукта: что
> считается готовым на каждой версии и какие критерии отделяют один релиз от
> другого. Документ относится к «матчасти» (справочные материалы) и
> поддерживается актуальным по мере развития платформы.
>
> **Опорные точки (по требованию issue):**
>
> - **1.0.0** — полностью готовый цифровой двойник: **весь функционал, все
>   контуры, все сценарии**.
> - **0.2** — **полностью рабочий открытый контур**.

## Принципы версионирования

- Используется семантическое версионирование `MAJOR.MINOR.PATCH`.
- **PATCH** (`0.1.x`) — улучшения и новые возможности внутри текущего этапа без
  смены границ контура: правки UI, новые слои и сценарии открытого контура,
  исправления, документация.
- **MINOR** (`0.x`) — закрытие крупного этапа дорожной карты (например, переход
  «каркас открытого контура» → «полностью рабочий открытый контур»).
- **MAJOR** (`1.0.0` и далее) — цель проекта: единый цифровой двойник со всеми
  контурами и сценариями в продуктивной эксплуатации.
- Все версии до `1.0.0` относятся **исключительно к открытому контуру** и
  демонстрационным/открытым данным. Ведомственные, инфраструктурные и закрытые
  контуры подключаются только после `0.2`, по мере межведомственных соглашений.

## Контуры доступа

Целевая модель — многоуровневая: **открытый**, **ведомственный**,
**инфраструктурный**, **закрытый**, **исследовательский** контуры обрабатываются
раздельно, с ролевым доступом, обезличиванием и аудитом. Дорожная карта
последовательно наращивает контуры от открытого к закрытому.

## Версии

### 0.1.x — Каркас открытого контура (текущий этап)

Демонстратор открытого контура на демо- и открытых данных.

- Каталог датасетов с паспортами (источник, версия, лицензия, качество,
  ограничения).
- Карта субъектов РФ (MapLibre GL), слои объектов, попапы с провенансом.
- Демо-сценарии с детерминированными результатами и экспортом (Markdown/PDF).
- Ролевой доступ (гость, гражданин, бизнес, разработчик, оператор), аудит
  действий с цепочкой хешей.
- **0.1.4 (этот релиз):**
  - Дорожная карта по версиям (этот документ).
  - Метрики готовности (датасеты, домены, сценарии, объекты карты) показываются
    только в разделе «Отчёты».
  - Выгрузка сырых данных (CSV) в каталоге — **только для операторов
    платформы**.
  - Переключение подложки на Яндекс Карты (при наличии API-ключа) с подсветкой
    регионов; по умолчанию — OSM и бесплатный геокодер Nominatim (≤1 запрос/с,
    контактный e-mail в `User-Agent`). Лимиты Яндекса для бесплатного плана:
    ≤1000 запросов/час и ≤950 запросов/сутки для HTTP-геокодера.
  - Первый слой данных — **вакансии «Работа России»**
    (`opendata.trudvsem.ru`): включается/выключается, комбинация кластеризации и
    тепловой карты, топ профессий в сайдбаре, поиск и фильтр по специальности,
    карточка вакансии при клике. Потоковая загрузка CSV без сохранения на диск,
    инкрементальное обновление по `modifiedFrom`, периодичность ~1,5 суток.

### 0.2 — Полностью рабочий открытый контур

**Критерий готовности версии:** открытый контур работает целиком, не как
демонстратор, а как практический инструмент на реальных открытых данных.

- Все ключевые открытые источники подключены через рабочий ETL (CSV → API/JSON),
  данные регулярно обновляются по расписанию, с паспортами и контролем качества.
- Полноценный геосервис: геокодирование адресов, привязка объектов к регионам и
  ОКТМО, слои с реальными координатами.
- Набор прикладных сценариев открытого контура (рынок труда, доступность
  инфраструктуры, ресурсная обеспеченность) на реальных данных, а не демо.
- Производительность и наблюдаемость: кэширование, метрики, журналы, лимиты
  внешних API соблюдаются автоматически.
- Публичный API с ключами и квотами; выгрузка датасетов для операторов.

### 0.3–0.5 — Ведомственный и инфраструктурный контуры

- Подключение ведомственных данных по межведомственным соглашениям, разделение
  контуров, обезличивание и k-анонимность для чувствительных датасетов.
- Инфраструктурные слои (сети, мощности, логистика) в защищённом контуре.
- Расширение ролевой модели и регламентов доступа, задержка/обобщение
  чувствительных данных.

### 0.6–0.9 — Закрытый и исследовательский контуры, межведомственные сценарии

- Закрытый контур: схемы критической инфраструктуры, резервные маршруты и
  цепочки снабжения в разрешённом доступе.
- Исследовательский контур: моделирование, прогнозы, сценарии «что если».
- Межведомственные сквозные сценарии, объединяющие несколько контуров.

### 1.0.0 — Полностью готовый цифровой двойник

**Критерий готовности версии:** единый цифровой двойник России со **всем
функционалом, всеми контурами и всеми сценариями** в продуктивной эксплуатации.

- Все контуры (открытый, ведомственный, инфраструктурный, закрытый,
  исследовательский) работают одновременно с корректным разделением доступа.
- Полный каталог сценариев национального ресурсного планирования.
- Эксплуатационная модель 24/7 для критичных контуров: резервирование,
  журналирование, контроль качества, аудит.
- Масштаб всей страны, а не отдельных пилотных регионов.
