# Подложка Яндекс Карт: рантайм-ключ и переменные `VITE_*`

## Почему кнопка «Яндекс» оставалась неактивной

Фронтенд собран на Vite. Переменные окружения с префиксом `VITE_`
(`VITE_YANDEX_API_KEY`, `VITE_API_BASE_URL`) **подставляются в бандл во время
сборки** (`npm run build`), а не читаются в рантайме. Поэтому:

- задать `VITE_YANDEX_API_KEY` в окружении уже запущенного контейнера
  бесполезно — бандл собран без ключа, и кнопка «Яндекс» остаётся отключённой;
- простой перезапуск контейнера (`docker restart` / `docker compose restart`)
  **не пересобирает** фронтенд, поэтому ключ так и не попадает в код.

Это и было причиной из issue #17 и снова из issue #21: «карта Яндекса по-прежнему
не открывается, кнопка неактивна, хотя в `.env` ключ есть».

## Как правильно включить Яндекс Карты (рекомендуется, issue #21)

Начиная с v0.1.5 ключ можно задать **на стороне бэкенда** — он отдаёт его
фронтенду в рантайме через эндпоинт `/api/v1/config`. Пересборка фронтенда при
этом **не требуется**, достаточно перезапуска контейнера.

1. Скопируйте пример окружения и впишите ключ в переменную **без** префикса
   `VITE_`:

   ```bash
   cp .env.example .env
   # отредактируйте .env:
   # YANDEX_API_KEY=ваш-ключ-яндекса
   ```

2. Поднимите (или перезапустите) стек:

   ```bash
   make up
   ```

   `infra/docker-compose.yml` пробрасывает `.env` в окружение бэкенда через
   `env_file`, поэтому `YANDEX_API_KEY` сразу доступен процессу FastAPI. Эндпоинт
   `GET /api/v1/config` возвращает `{"yandex_api_key": "...", "yandex_enabled":
   true}`, фронтенд читает его при загрузке карты и активирует кнопку «Яндекс».

3. Если ключ задан, но кнопка неактивна — проверьте ответ эндпоинта напрямую:

   ```bash
   curl http://localhost:8000/api/v1/config
   ```

   Пустой `yandex_api_key` означает, что переменная не дошла до бэкенда (опечатка
   в имени, файл `.env` не подхвачен и т. п.).

## Устаревший способ: build-time `VITE_YANDEX_API_KEY`

Старый путь через build-arg по-прежнему работает как запасной (фронтенд берёт
build-time ключ, если рантайм-конфиг пуст), но требует **пересборки**:

```bash
cp .env.example .env
# VITE_YANDEX_API_KEY=ваш-ключ-яндекса
make up      # эквивалент: docker compose -f infra/docker-compose.yml up --build
```

`make up` всегда вызывает `up --build`, поэтому образ пересобирается, и ключ из
`.env` попадает в бандл через build-arg. Если контейнеры уже запущены и вы только
что добавили ключ — недостаточно `make down && make up` без пересборки; используйте
именно `make up` (с `--build`) или явно
`docker compose -f infra/docker-compose.yml build frontend`. Предпочитайте
рантайм-переменную `YANDEX_API_KEY` выше — она избавляет от пересборки.

## Как ключ доходит до бандла (build-time путь)

`infra/docker-compose.yml` пробрасывает переменные в сборку как build-args:

```yaml
frontend:
  build:
    context: ..
    dockerfile: frontend/Dockerfile
    args:
      VITE_YANDEX_API_KEY: ${VITE_YANDEX_API_KEY:-}
      VITE_API_BASE_URL: ${VITE_API_BASE_URL:-}
```

`frontend/Dockerfile` объявляет соответствующие `ARG`/`ENV` перед
`npm run build`:

```dockerfile
ARG VITE_YANDEX_API_KEY=""
ARG VITE_API_BASE_URL=""
ENV VITE_YANDEX_API_KEY=$VITE_YANDEX_API_KEY
ENV VITE_API_BASE_URL=$VITE_API_BASE_URL
RUN npm run build
```

Значения `${VITE_YANDEX_API_KEY:-}` Docker Compose берёт из файла `.env` в
корне репозитория (или из переменных окружения текущей сессии). Если ключ не
задан — подложка остаётся OSM, а кнопка «Яндекс» отключена с подсказкой.

## Что писать в `VITE_API_BASE_URL`

`VITE_API_BASE_URL` — это адрес **нашего бэкенда** (FastAPI), а не внешнего
сайта. Сюда **не нужно** вписывать `https://yandex.ru` или любой другой внешний
ресурс (issue #19: «не знаю, что писать, поставил `https://yandex.ru`»).

- **Оставьте пустым** при штатном запуске через `make up`. Фронтенд и бэкенд
  доступны на одном origin, а nginx внутри контейнера фронтенда проксирует все
  запросы `/api/...` на бэкенд. Пустое значение означает «тот же origin», и в
  коде запрос идёт как `fetch("/api/v1/...")` — браузер сам подставит текущий
  адрес сайта. Это правильный режим для большинства пользователей.
- **Заполняйте только если** фронтенд и бэкенд работают на **разных адресах**.
  Типичный случай — локальная разработка фронтенда (`npm run dev` на
  `http://127.0.0.1:5173`) отдельно от бэкенда:

  ```bash
  VITE_API_BASE_URL=http://127.0.0.1:8000
  ```

  В этом режиме бэкенд должен разрешать CORS для адреса фронтенда (по умолчанию
  в `backend/app/main.py` разрешены `http://localhost:5173` и
  `http://127.0.0.1:5173`).

Как и `VITE_YANDEX_API_KEY`, эта переменная «запекается» в бандл во время
сборки, поэтому после её изменения фронтенд нужно пересобрать (`make up`).

## Лимиты бесплатного плана Яндекса

HTTP-геокодер Яндекса в бесплатном плане ограничен ~1000 запросов/час и
~950 запросов/сутки. Поэтому геокодирование адресов вакансий по умолчанию идёт
через бесплатный Nominatim (≤1 запрос/с), а подложка Яндекс Карт подключается
только для отображения тайлов при наличии ключа.
