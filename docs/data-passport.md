# Контракт паспорта датасета

Каждый датасет открытого контура представлен паспортом в
`data/demo/datasets.json`.

Обязательные поля:

| Поле                    | Назначение                                                       |
| ----------------------- | ---------------------------------------------------------------- |
| `id`                    | Стабильный идентификатор датасета                                |
| `title`                 | Название, понятное человеку                                      |
| `domain`                | Домен данных для фильтрации                                      |
| `region`                | Пилотный регион или национальный охват                           |
| `owner`                 | Владелец или распорядитель данных                                |
| `source` / `source_url` | Источник и точка доступа                                         |
| `source_version`        | Дата выгрузки или снимка                                         |
| `license`               | Условия публикации или повторного использования                  |
| `update_frequency`      | Ожидаемая периодичность обновления                               |
| `classifier_alignment`  | Привязка к ОКТМО/ОКВЭД/ОКЗ/ОКСО/WGS-84                         |
| `pii_status`            | `none`, `aggregated`, `anonymized` или `pseudonymized`           |
| `k_anonymity`           | Защита агрегации для публичного контура                          |
| `known_limitations`     | Явные оговорки, отображаемые в отчётах                           |
| `validators`            | Проверки, применяемые ETL или при ревью                          |
| `quality_flag`          | `verified`, `aggregated`, `draft` или `outdated`                 |
| `contour`               | В v0.1 должно быть `open`                                        |

Запуски сценариев и экспорты включают поля source/version/license/quality из
этих паспортов, чтобы рецензенты могли отследить каждую метрику до исходного
каталога.
