# Есть ли у Яндекс Карт собственный слой с обозначением регионов?

**Короткий ответ: да.** У Яндекс Карт есть официальный слой границ
административных регионов, но он доступен только через **Yandex Maps JavaScript
API 2.1** — модуль `ymaps.borders`. В нашем прототипе подложка Яндекса
подключается как **растровые тайлы** внутри MapLibre GL, поэтому этот слой у нас
«из коробки» недоступен: названия и границы регионов на тайлах Яндекса
нарисованы как картинка (пиксели), а не как отдельный запрашиваемый
векторный слой.

## Что именно даёт Яндекс

### 1. Растровые тайлы (то, что используем мы)

Подложка Яндекс Карт, которую фронтенд подключает по ключу `YANDEX_API_KEY`
(см. [docs/yandex-basemap.md](yandex-basemap.md)), уже содержит подписи и границы
субъектов РФ — но «вплавленными» в изображение тайла. Их видно, однако нельзя
кликнуть по региону, подсветить его полигон или получить его код/название
программно. Для нашей карты этого достаточно как фона.

### 2. Векторный слой границ регионов (`ymaps.borders`)

Полноценный слой с геометрией регионов есть в Yandex Maps JavaScript API 2.1.
Функция `ymaps.borders.load(countryCode, options)` возвращает `Promise`,
который разрешается объектом **GeoJSON** с полигонами регионов. Пример:

```js
ymaps.borders
  .load("RU", { lang: "ru", quality: 2 })
  .then((geojson) => {
    // geojson.features — полигоны субъектов РФ
    // у каждого feature.properties есть: name, iso3166, level, hintContent
  });
```

Параметры:

- `lang` — язык подписей (ISO 639-1);
- `quality` — детализация границ, `0…3` (чем выше, тем точнее и «тяжелее»);
- `disputedBorders` — трактовка спорных границ (`RU`, `UA`, `UN`).

Полученные полигоны можно раскрасить (хороплет), навесить обработчики кликов,
показать подписи — это и есть «слой с обозначением регионов».

## Почему у нас его пока нет и как добавить

Мы рисуем карту через **MapLibre GL**, а не через Yandex Maps JS API, поэтому
`ymaps.borders` напрямую не подключить. Варианты, если понадобится
интерактивный слой регионов:

1. **Взять границы из открытых источников.** Загрузить полигоны субъектов РФ как
   GeoJSON (например, из OpenStreetMap / GADM / модуля
   [`esosedi/regions`](https://github.com/esosedi/regions), ~300 000 регионов на
   ru/en/de) и отрисовать их отдельным слоем MapLibre (`fill` + `line`). Это не
   требует ключа Яндекса и работает с любой подложкой.
2. **Перейти на Yandex Maps JS API 2.1** для карты целиком и использовать
   `ymaps.borders.load('RU', …)`. Это меняет весь рендер-стек карты и привязывает
   нас к ключу и лимитам Яндекса.

**Вывод:** собственный слой обозначений регионов у Яндекса есть
(`ymaps.borders` в JS API 2.1); в текущем прототипе на MapLibre мы используем
только растровую подложку Яндекса, а интерактивные полигоны регионов при
необходимости проще добавить из открытого GeoJSON отдельным слоем MapLibre.
